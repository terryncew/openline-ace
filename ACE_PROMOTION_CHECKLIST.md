# ACE Promotion Checklist

## Level 0 → Level 1

- [ ] Candidate structure identified
- [ ] Search settings recorded
- [ ] Initial receipt emitted
- [ ] Basic control run

## Level 1 → Level 2

- [ ] Random controls passed
- [ ] Nearby-parameter controls passed
- [ ] Window sensitivity checked
- [ ] Held-out prediction passed
- [ ] Measurement-frame audit run
- [ ] Failure modes recorded

## Level 2 → Level 3

- [ ] Exact definitions written
- [ ] Finite certificate manifest generated
- [ ] Falsifiers listed
- [ ] Proof obligations listed
- [ ] Formalization status assigned
- [ ] Ledger entry added

## Level 3 → Level 4

- [ ] Formal definitions machine-checked or reviewed
- [ ] Finite certificates verified
- [ ] Lemmas stated precisely
- [ ] Theorem or finite claim proven
- [ ] Receipt chain links proof to evidence

## Hard Rule

A claim cannot promote itself.

Explorer proposes.

Auditor attacks.

Formalizer packages.

Outside witness or independent verification decides promotion.
